# Shipping Tickets Snippet

A Chrome extension template (Manifest V3) built with Vue 3 + Vite + TypeScript + [@crxjs/vite-plugin](https://crxjs.dev/vite-plugin).

Clicking the extension icon opens the settings page (`index.html`) with **API token / Email / Summary** fields; the values are persisted to `chrome.storage.local`.

## Structure

```
.
├── public/
│   └── icons/                # PNG icons 16/48/128 (copied to dist/ as-is)
├── src/
│   ├── background.ts         # service worker
│   ├── shims-vue.d.ts        # types for .vue imports
│   └── ui/
│       ├── main.ts           # Vue bootstrap
│       ├── App.vue           # form (Composition API + <script setup>)
│       └── style.css         # global styles
├── index.html                # extension page (Vite entry)
├── manifest.config.ts        # typed manifest (generated by the plugin)
├── vite.config.ts
├── tsconfig.json
└── package.json
```

## Install

```bash
npm install
```

## Development with HMR

```bash
npm run dev
```

Vite starts a dev server and rebuilds `dist/` at the same time. Load the extension into Chrome once:

1. `chrome://extensions/` → enable **Developer mode**
2. **Load unpacked** → pick the `dist/` folder

After that:
- editing [src/ui/main.ts](src/ui/main.ts) or [index.html](index.html) → HMR on the extension page, no reload needed
- editing [src/background.ts](src/background.ts) or [manifest.config.ts](manifest.config.ts) → the plugin asks Chrome to reload the extension automatically

## Production build

```bash
npm run build
```

The resulting bundle is in `dist/`. You can load that folder as an unpacked extension or pack it via `chrome://extensions/` → **Pack extension**.

## Icons

Put three files into [public/icons/](public/icons/): `icon16.png`, `icon48.png`, `icon128.png`. The extension still loads without them — Chrome just shows a placeholder icon.

## Changing the URL / behavior

- Form fields and persistence logic — [src/ui/App.vue](src/ui/App.vue).
- Extension icon click action — [src/background.ts](src/background.ts).
- Name, description, permissions — [manifest.config.ts](manifest.config.ts).
